🔔 Notification Behaviour
📖 Chapter updates → Pushover
When a new chapter is detected via MangaUpdates, send a Pushover push to your device. This is the main notification type — recommended to keep on.
📰 News items → Pushover
News articles about tracked series also appear in-app. Enable this only if you want them pushed to your phone too — can be noisy.
📲 Push only for "Reading" status
When enabled, Pushover notifications only fire for series you have marked Reading. Series set to On Hold, Plan to Read, Considering, etc. still create in-app notifications but won't ping your device. Useful if you have a large backlog you haven't started yet.
🔑 Pushover Credentials
One-time $5 purchase at pushover.net. Works on iOS & Android.
📡 MangaUpdates Integration
When enabled, the poller uses MangaUpdates' release feed for accurate chapter-level notifications including release group names. MangaUpdates requires no API token. Falls back to MangaBaka chapter counts for unlinked series.
🔄 Update Polling
🔵 K Manga / Kodansha

K Manga now requires a reCAPTCHA v3 token when logging in fresh. You only need this if the session expires. Steps: log in at kmanga.kodansha.com in your browser, open DevTools → Network, find the login POST request, copy the g-recaptcha-response field value, paste it here, save, then click Poll Now. The token is consumed once and cleared automatically.

Used to poll official Kodansha simulpub releases via the K Manga API. US-based IP required. Credentials are stored locally and never sent anywhere except K Manga's own servers.
🔑 MangaBaka API
Token from mangabaka.org. Rate limits: 30 searches/min, 120 lookups/min.
