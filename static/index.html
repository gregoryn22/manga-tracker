🔔 Notification Behaviour
📖 Chapter updates → Pushover
When a new chapter is detected via MangaUpdates, send a Pushover push to your device. This is the main notification type — recommended to keep on.
📰 News items → Pushover
News articles about tracked series also appear in-app. Enable this only if you want them pushed to your phone too — can be noisy.
📲 Push only for "Reading" status
When enabled, Pushover notifications only fire for series you have marked Reading. Series set to On Hold, Plan to Read, Considering, etc. still create in-app notifications but won't ping your device. Useful if you have a large backlog you haven't started yet.
⚠ Polling failure alerts → Pushover
Send a Pushover notification when a series reaches the consecutive failure threshold below. Fires once when the threshold is hit, then again each time it doubles (e.g. 5, 10, 20…).
🟢 Dashboard "Updates" filter
When set to "Reading only", the Updates pill and badge on the dashboard only count series with Reading status. Plan to Read, On Hold, etc. with new chapters still show a NEW chip on their card — they just won't be counted in the update total.
🔑 Pushover Credentials
One-time $5 purchase at pushover.net. Works on iOS & Android.
📡 MangaUpdates Integration
When enabled, the poller uses MangaUpdates' release feed for accurate chapter-level notifications including release group names. MangaUpdates requires no API token. Falls back to MangaBaka chapter counts for unlinked series.
🔄 Update Polling
🔵 K Manga / Kodansha

K Manga now requires a reCAPTCHA v3 token when logging in fresh. You only need this if the session expires. Steps: log in at kmanga.kodansha.com in your browser, open DevTools → Network, find the login POST request, copy the g-recaptcha-response field value, paste it here, save, then click Poll Now. The token is consumed once and cleared automatically.

Used to poll official Kodansha simulpub releases via the K Manga API. US-based IP required. Credentials are stored locally and never sent anywhere except K Manga's own servers.
📚 Komga (Self-Hosted)
Connect your Komga server to track chapter updates from your self-hosted library. Generate an API key in Komga: User Settings → API Keys.
💤 Idle Series Detection
When enabled, series with no releases detected in the specified number of days will be flagged with a 💤 chip. Useful for spotting series that may have gone on hiatus or been dropped by scanlators. If "Move to Dropped" is selected, they will be automatically set to Dropped status at the end of each poll cycle. Only applies to non-simulpub series.
🔗 Discord / Slack Webhook
Supports Discord webhook URLs and Slack incoming webhook URLs. Chapter notifications will be sent alongside Pushover (if enabled).
💾 Library Backup
Export your entire library as a JSON file for backup or migration. Import will skip series that are already tracked.
🔄 MangaBaka Sync
Export your library in MangaBaka's native format. Import the file at mangabaka.org/my/library/import to sync reading progress and status to your MangaBaka account.
🔧 Metadata Maintenance
Fetches cover art from MangaBaka for any series missing a cover image. Runs in the background — refresh the library after a moment to see results.
🎨 UI Customization
Customize your dashboard experience. Changes are applied immediately.
🃏 Library Card Display
Hiding elements reduces visual noise — useful with compact grid density.
🔑 MangaBaka API
Token from mangabaka.org. Rate limits: 30 searches/min, 120 lookups/min.
